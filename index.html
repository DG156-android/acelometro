MPU6050×ESP32

Dos formas de leer el mismo sensor: una con librería, otra hablando directo al registro I2C.

01 — conexión

Cuatro cables, un bus I2C

3V3 → VCC GND → GND GPIO21 → SDA GPIO22 → SCL
  • Alimentación — el MPU6050 corre a 3.3–5V; con el ESP32 se usa la salida de 3V3.
  • I2C — SDA y SCL son las únicas líneas de datos; el sensor responde en la dirección 0x68.
  • Un solo bus — se pueden colgar varios sensores I2C en los mismos dos cables.
02 — Arduino (C++)

La vía cómoda: librería Adafruit

#include <Wire.h>
#include <Adafruit_MPU6050.h>
#include <Adafruit_Sensor.h>

Adafruit_MPU6050 mpu;

void setup() {
  Serial.begin(115200);
  while (!Serial) delay(10);

  Serial.println("Iniciando MPU6050...");

  if (!mpu.begin()) {
    Serial.println("No se encontró el sensor. Revisa las conexiones.");
    while (1) delay(10);
  }
  Serial.println("MPU6050 encontrado!");

  mpu.setAccelerometerRange(MPU6050_RANGE_8_G);
  mpu.setGyroRange(MPU6050_RANGE_500_DEG);
  mpu.setFilterBandwidth(MPU6050_BAND_21_HZ);

  delay(100);
}

void loop() {
  sensors_event_t a, g, temp;
  mpu.getEvent(&a, &g, &temp);

  Serial.print("Accel X: "); Serial.print(a.acceleration.x);
  Serial.print(" | Y: "); Serial.print(a.acceleration.y);
  Serial.print(" | Z: "); Serial.println(a.acceleration.z);

  Serial.print("Gyro  X: "); Serial.print(g.gyro.x);
  Serial.print(" | Y: "); Serial.print(g.gyro.y);
  Serial.print(" | Z: "); Serial.println(g.gyro.z);

  Serial.println("------------------------------------");
  delay(500);
}
01 · librerías

Wire.h maneja el bus I2C a bajo nivel; Adafruit_MPU6050 y Adafruit_Sensor lo envuelven en funciones listas para usar, sin tocar registros a mano.

02 · setup()

Abre el puerto serie a 115200 baudios y espera a que esté listo. Llama a mpu.begin(); si no encuentra el sensor, queda atrapado en while(1) avisando el error para siempre.

03 · configuración

Tres ajustes clave: rango del acelerómetro ±8G (más rango, menos precisión fina), rango del giroscopio ±500°/s, y un filtro pasa-bajos de 21Hz que suaviza vibraciones y ruido eléctrico.

04 · loop()

mpu.getEvent(&a,&g,&temp) hace toda la lectura I2C internamente y devuelve tres structs ya en unidades físicas (m/s², rad/s). Se imprimen los 6 ejes y se repite cada 500ms.

03 — MicroPython

La vía directa: registros crudos

from machine import Pin, I2C
import time

i2c = I2C(0, scl=Pin(22), sda=Pin(21), freq=400000)
MPU_ADDR = 0x68

def mpu_init():
    i2c.writeto_mem(MPU_ADDR, 0x6B, b'\x00')  # Despierta el sensor

def read_word(reg):
    high = i2c.readfrom_mem(MPU_ADDR, reg, 1)[0]
    low = i2c.readfrom_mem(MPU_ADDR, reg + 1, 1)[0]
    value = (high << 8) | low
    if value >= 0x8000:
        value -= 0x10000
    return value

def get_data():
    ax = read_word(0x3B) / 16384.0
    ay = read_word(0x3D) / 16384.0
    az = read_word(0x3F) / 16384.0
    gx = read_word(0x43) / 131.0
    gy = read_word(0x45) / 131.0
    gz = read_word(0x47) / 131.0
    return ax, ay, az, gx, gy, gz

mpu_init()
time.sleep(0.5)

while True:
    ax, ay, az, gx, gy, gz = get_data()
    print("Accel X:{:.2f}g Y:{:.2f}g Z:{:.2f}g".format(ax, ay, az))
    print("Gyro  X:{:.2f}°/s Y:{:.2f}°/s Z:{:.2f}°/s".format(gx, gy, gz))
    print("-" * 40)
    time.sleep(0.5)
01 · I2C manual

Se crea el bus I2C indicando explícitamente los pines: SCL en GPIO22, SDA en GPIO21, a 400kHz. Sin librería de sensor, solo el driver I2C del propio MicroPython.

02 · mpu_init()

El MPU6050 arranca en modo "sleep". Escribir 0x00 en el registro de gestión de energía 0x6B lo despierta y empieza a muestrear.

03 · read_word()

Cada eje ocupa dos registros (byte alto y bajo). Se combinan con desplazamiento de bits (high<<8)|low y se convierte a complemento a dos si el valor es negativo.

04 · get_data()

Lee los seis registros crudos (0x3B–0x47) y los escala a unidades reales: entre 16384 para acelerómetro (±2g) y entre 131 para giroscopio (±250°/s).

05 · bucle principal

Inicializa el sensor, espera 0.5s a que estabilice, y luego imprime las 6 lecturas cada medio segundo indefinidamente.
